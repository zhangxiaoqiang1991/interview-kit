# 四步法完整教程

## 问题：为什么大多数面试准备是低效的

典型面试准备是这样做的：

1. 拿到 JD，把每条要求都看一遍
2. 每条要求准备一个回答
3. 找 30 道常见面试题，准备 30 个答案
4. 背公司介绍、背创始人语录

问题在哪？一场面试 30-45 分钟。面试官有评估表，但人的注意力是有限的漏斗：

- 前 15-20 分钟：深挖 JD 第 1 条（占 50-60% 时间）
- 接下来 10-15 分钟：快速过 JD 第 2-3 条
- 最后 5 分钟：你问他问题

你花了 2 小时准备的 30 道题，面试官根本没问到。他甚至可能没看到 JD 上的第 5 条。

**你的准备时间应该精确 match 面试官的时间分配。**

---

## 第一步：JD 权重解码

### 方法

拿到 JD，做三件事：

1. **标序号**：把 JD 上的每一条要求从 1 到 N 编号
2. **区分职责和要求**：职责 = 「你要做什么」，要求 = 「你得有什么」。**要求的序号比职责更重要**
3. **按序号分配时间**：

```
#1 → 50-60% 准备时间 → 准备深度答案（能扛 3 层追问）
#2 → 20-30% 准备时间 → 准备完整案例 + 1 层追问
#3 → 10-15% 准备时间 → 准备一个能说的亮点
#4+ → 扫一眼即可 → 如果你看 #4 完全不相关，不用担心
```

### 实例

一份典型的**B2B SaaS 增长负责人** JD：

```
岗位职责：
1. 制定并执行用户增长策略，对用户规模和活跃度负责
2. 搭建内容营销体系，通过内容驱动 inbound 获客
3. 协同销售团队，提升线索到成交的转化效率
4. 建立增长数据看板，驱动数据化决策
5. 管理增长团队（3-5 人）

任职要求：
1. 5 年以上 B2B SaaS 增长经验
2. 有成功的内容营销案例
3. 熟悉数据分析工具（SQL、Mixpanel 等）
4. 有团队管理经验
5. 英语流利
```

**权重解码结果：**

| 序号 | 内容 | 面试时间占比 | 你的准备时间 |
|:---:|------|:---:|:---:|
| #1 | 增长策略 + 对规模和活跃度负责 | 50-60% | 60 分钟 |
| #2 | 内容营销体系 + inbound 获客 | 20-30% | 30 分钟 |
| #3 | 协同销售 + 转化效率 | 10-15% | 15 分钟 |
| #4 | 数据看板 | 可能问一句 | 5 分钟 |
| #5 | 团队管理 | 可能不问 | 扫一眼 |

对于 #1，准备一个完整案例（你曾经从 0 搭建增长体系的经历），能扛住「你怎么定的北极星指标」「失败过什么实验」「如果重来会怎么改」三层追问。

对于 #5，准备好「我带过 X 人团队，主要方式是每周 1-1 + 季度 OKR」一句话就够了。

**不要花同样的时间准备 #1 和 #5。**

## 第二步：核心题预判

### 方法

JD 第 1 条 = 面试官最想验证的能力。他的问题不是「你做没做过」，是三个层次：

#### 第一层：你做过（验证真实性）

典型问法：
- 「讲讲你在 XX 方面的经验」
- 「你做过的最相关的项目是什么」

**准备：** 一个 2 分钟的 STAR 案例（Situation → Task → Action → Result）。关键词：你的角色、具体动作、数字结果。不要讲「我们团队做了 XX」——面试官要听的是「你」做了什么。

#### 第二层：你理解（验证深度）

典型问法：
- 「你当时遇到了什么困难？」
- 「你觉得当时有什么可以改进的？」
- 「你的 KPI 是怎么定的？为什么这样定？」

**准备：** 一个你克服的具体困难。选一个真实的问题——不要用「跨团队沟通」这种万能答案。比如：「最大的困难是增长实验的样本量不够。我们只有 2000 个用户，跑 A/B 测试统计显著性达不到。后来我们改用了时间序列对比，不是完美的方案，但在那个阶段是可行的。」

#### 第三层：你能迁移（验证适应性）

典型问法：
- 「来我们这里，同样的方法能直接用吗？」
- 「你觉得我们面临的最大的增长挑战是什么？」
- 「如果给你 3 个月，你会怎么开始？」

**准备：** 一个从旧到新的迁移思路。不要假设方法论可以直接搬。先说区别：旧公司和新公司在用户规模、资源、市场阶段的差异，再说你会调整什么，保留什么。

### 覆盖度计算

如果 JD 前三条都跑这个三问：

```
#1: 提问 A1, A2, A3  → 准备 3 个回答片段
#2: 提问 B1, B2      → 准备 2 个回答片段（第三层可能不问）
#3: 提问 C1           → 准备 1 个回答片段
```

总共准备 6 个回答片段。这 6 个片段覆盖 80% 的面试内容。比准备 30 道题有效。

## 第三步：提问即信号

### 方法

面试进行中，每听到一个问题，立刻映射：他在验证 JD 的第几条？

**练习：** 面试前一天，把你的 JD 拿过来，每一条旁边写了一行：「面试官可能怎么问」。

| JD 要求 | 可能的问题 |
|---------|-----------|
| 「5 年以上 B2B SaaS 增长经验」 | 「讲讲你最成功的一次增长实验」 |
| 「有成功的内容营销案例」 | 「内容营销里你最大的杠杆点是什么？」 |
| 「熟悉数据分析工具」 | 「你是怎么用数据驱动决策的？举个例子」 |

面试时，如果面试官连「内容营销」都没提到，就说明这条在 JD 上不是真的核心——不管 JD 怎么写。

**反向使用：** 如果你发现面试官一直在问 JD #3 的内容，而不怎么问 #1——这可能意味：
- 他对 #1 的答案已经满意了（好事）
- 或者 #1 不是你来做（岗位实际情况跟 JD 不太一样，需要留意）
- 或者他发现你 #1 的亮点不明显，转而去验证你其他方面能不能加分

## 第四步：被纠正 = 还在局里

### 信号解读

面试中面试官的语言分三个层次：

| 信号 | 含义 | 你的行动 |
|------|------|---------|
| **深度追问** | X 他感兴趣，想确认你是真懂 | 展开细节，但要收得住 |
| **纠正你/拉回来** | X 你偏了，但他不想放弃你 | 停下来，承认偏了，重新对准 |
| **沉默 /「好的了解了」** | X 他已做出判断，不再投入 | 尝试切换话题，看能不能重新激活 |
| **开始聊公司/团队** | X 他已经在想你能不能融入 | 展示真实的一面，不要继续面试模式 |
| **提前开始卖公司** | X 他想要你了 | 认真听，提好问题 |

### 被纠正时的正确做法

❌ 不要解释 — 「我的意思是...」「殊途同归...」「其实本质上一样...」

✅ 停下来，承认 — 「你说的对，我刚才偏了。我重新来。」

纠正你 = 投人。不纠正你 = 放弃你。面试官纠正你，是他还把你当成可能的候选人。

## 复习：15 分钟速查表

面试当天早上，花 15 分钟过一遍：

```
[ ] Step 1: JD 标序号（2 分钟）
     #1 = ________, #2 = ________, #3 = ________
     #1 我要花 _____ 分钟讲

[ ] Step 2: 准备 6 个回答片段（8 分钟）
     #1 三层追问：案例 / 困难 / 迁移
     #2 两层：案例 / 困难
     #3 一层：亮点

[ ] Step 3: 提问映射（3 分钟）
     每条 JD 写一个可能的问题

[ ] Step 4: 心理准备（2 分钟）
     被纠正 = 好信号
     被追问 = 好信号
     沉默 = 需要切换话题
     面试官提前开始卖公司 = 想要你了
```
