# 案例一：B2B SaaS 增长负责人

- **公司阶段**：A 轮，50 人，ARR 200 万美元
- **行业**：企业协作 SaaS
- **候选人背景**：前字节跳动用户运营，B2C 转 B2B
- **关键挑战**：从「海量用户 + 内容运营」的经验，切换到「精准线索 + 内容驱动 inbound」的模式

## JD 原貌

```
岗位：增长负责人  |  汇报：CEO

职责：
1. 制定并执行用户增长策略，对 PLG 漏斗全链路负责
2. 搭建内容营销体系，通过内容驱动 inbound 获客
3. 协同销售团队，提升线索到成交的转化效率
4. 建立增长数据看板，驱动数据化决策
5. 管理 2-3 人增长团队

要求：
1. 5 年以上增长/运营经验，有 B2B SaaS 经验优先
2. 有成功的内容营销案例（博客、白皮书、SEO 等）
3. 熟练使用数据分析工具（SQL、Mixpanel 或 Amplitude）
4. 有团队管理经验
```

## Step 1：JD 权重解码

| 序号 | 内容 | 面试时间 | 准备优先级 |
|:---:|------|:---:|:---:|
| #1 | PLG 漏斗全链路增长策略 | 60% | X 最高 |
| #2 | 内容营销体系 + inbound 获客 | 25% | X 高 |
| #3 | 协同销售 + 转化效率 | 10% | X 中 |
| #4 | 数据看板 | 5% | X 低 |
| #5 | 团队管理（2-3 人） | 可能不提 | X 最低 |

**关键发现：** 这个岗位看起来要求「全链路增长」，但 CEO 亲自面的种子轮公司，他真正关心的是第一条——你有没有从 0 搭过增长体系。内容营销、销售协同、数据看板都是体系搭好之后的产物。**面试的核心题不是「你怎么做内容营销」，而是「你怎么从 0 搭增长体系」。**

## Step 2：核心题预判

### #1 三层追问准备

**第一层：你做过**
> 面试官：「讲讲你在增长方面最有代表性的经历。」

**准备方向：** 不是讲你在大厂的运营项目——面试官知道你在大厂不缺流量。你要讲的是**从有限资源里出结果**的经历。比如你曾经用 0 预算跑了一个小实验，从 100 个种子用户里筛出了增长信号。

**第二层：你理解**
> 面试官：「你当时遇到的增长瓶颈是什么？怎么破的？」

**准备方向：** 选一个方法论层面的困难。不要说「流量不够」。要说「用户来了但是不转化，我发现是 onboarding 流程有 3 个卡点，改了以后注册到激活的转化率从 20% 提到了 40%」。

**第三层：你能迁移**
> 面试官：「如果让你来搭我们的增长体系，前三个月你会做什么？」

**准备方向：** 不要说「我之前的经验可以直接用」——这家公司是 PLG 导向的 SaaS，跟大厂的运营逻辑不同。说：「我要先搞清楚两件事：第一，你们现在的用户是怎么来的，口碑还是投放？第二，用户激活的标准是什么——是注册了就算，还是用了核心功能才算？搞清楚了这两个问题，我才知道增长的瓶颈在哪。」**面试官要的不是你的方案，是你的第一步很清醒。**

### #2 内容营销两层准备

**第一层：**
> 面试官：「内容驱动 inbound 获客，你有什么思路？」

不是聊内容调性。是聊：内容选题怎么匹配获客意图、内容分发渠道怎么选、怎么衡量内容 ROI。

**第二层：**
> 面试官：「内容营销你踩过什么坑？」

别说「内容质量不够高」。说：「我花了三个月写了一堆文章，发现大部分流量来自技术类内容，但技术类内容的读者很难转化。后来我们分了内容线：用技术内容引流，用案例白皮书转化。」

## Step 3：提问即信号

| 如果面试官一直问... | 信号 | 你的策略 |
|:--|:--|:--|
| 增长体系和实验方法 | X 他是增长背景出身，要深挖 | 讲方法论，拉出数据 |
| 内容选题和 SEO | X 他不懂增长，只是知道要内容营销 | 把内容连接到增长指标 |
| 销售协同 | X 增长团队可能会跟销售有摩擦 | 强调过往跟销售配合的经验 |
| 团队怎么招人 | X 他关心你能不能自己拉起团队 | 讲你的 hiring 逻辑和 1 年内的组织规划 |

## Step 4：关键信号

这家公司的 CEO 亲自面增长负责人，说明增长是公司当前第一要务。你的自我介绍的权重分配：**60% 时间讲你从 0 搭体系的经历，30% 讲内容营销理解，10% 提一下你管过小团队。**

自我介绍不要说「我在字节做了 4 年运营，日活几千万」。因为对方会想：「那你来我们这儿，50 个人，几千用户，能适应吗？」——说「我最近半年在帮一个朋友从 0 做到 3000 注册用户」，即使这是一个 side project，比大厂数据更有说服力。
